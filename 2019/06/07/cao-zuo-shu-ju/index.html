SQL语句的分类

            DQL：数据查询语言              
            DML：数据操作语言          
            DDL：数据定义语言          
            DCL：数据控制语言        
            TVL：事务控制语言

插入语句 INSERT

INSERT 一次只能向表中插入一条数据 

INSERT INTO departments(department_id,department_name,manager_id,location_id)
VALUE (70,'public Relations',100,1700)

指定列名时，值的顺序一定要跟列的顺序一致
不指定列名时，值的顺序要和表中的顺序一致 

INSERT INTO departments
VALUES (72,'public Relations',100,1700)

可以往列中插入默认值，注意：必须保证该列允许空值

INSERT INTO departments
VALUES (74,'Public Relatios',DEFAULT,DEFAULT);

在插入时，如果指定列名，则一定要把所有非空列列举出来

当把记录插入到表中时，新的记录在表中的位置是不固定的。插入的记录在表中是无序的。

在插入记录时，注意不要主键约束，外键约束等五个约束

从其他表中拷贝数据
在 INSERT 语句中加入子查询。
INSERT INTO copy_emp
  SELECT *
  FROM   employees
  WHERE  job_id LIKE '%REP%';
不必书写 VALUES 子句。 
子查询中的值列表的个数与数据类型应与 INSERT 子句
中的列的个数和数据类型对应。

将查询语句结果一次性插入到表中，要求表的结构与查询结果的结构一致

INSERT IMTO copy
SELECT * 
FROM employees

更新数据 UPDATE

语法
UPDATE table 
SET column = value [,column=value,...]
[WHERE condition]

使用where子句制定需要更新的数据

UPDATE employees
SET department_id = 70 
WHERE employee_id=113;

注意：在使用UPDATE语句时，一定要加上WHERE子句

如果省略WHERE子句，则表中的所有数据都将被更新

UPDATE     copy_emp
SET        department_id = 110;

在UPDATE语句中也可以使用子查询

UPDATE  copy_emp
SET     department_id  =  (SELECT department_id
                           FROM employees
                           WHERE employee_id = 100)
WHERE   job_id         =  (SELECT job_id
                           FROM employees
                           WHERE employee_id = 200);

删除数据 DELETE

DELETE [FROM]      table
[WHERE      condition];

使用where子句删除指定记录 

DELETE FROM departments
 WHERE  department_name = 'Finance';

省略where子句则删除整表 

DELETE FROM  copy_emp;

删除数据注意完整性错误 不能删除正在被其他表使用的记录

显式默认值概述

使用 DEFAULT 关键字表示默认值
符合SQL:1999标准
可以使用显示默认值控制默认值的使用
显示默认值可以在 INSERT 和 UPDATE 语句中使用

在插入操作中使用默认值:

INSERT INTO departments
  (department_id, department_name, manager_id) 
VALUES (300, 'Engineering', DEFAULT);

数据库事务

事务是由完成若干工作的DML语句组成的逻辑单位

事务的四个特性：ACID
原子性(A)不可分割
一致性(C)事物开始前和结束后看到的都是一样的
隔离性(I)在事务进行中其他人是看不到的
永久性(D)事务结束后数据永远保存在数据库里面

数据库事务

一个或多个DML操作
一个DDL定义
一个DCL空值 

以第一个DML语句的执行作为开始  
以下面其中之一作为结束           
显示的COMMIT 或 ROLLBACK 语句            
DDL 或 DCL 语句（自动提交）          
用户退出iSQL*PLUS，会话正常结束         
系统异常终了

COMMIT和ROLLBACK语句的优点

事务结束时要么执行commit 要么执行rollback
确保数据的完整性
在数据改变被提交之前重新检查我们所改变的数据是否正确
将逻辑相关的操作分组

隐式事务进程

自动提交自动回滚的过程

自动提交在以下情况中执行:
DDL 语句。
DCL 语句。
不使用 COMMIT 或 ROLLBACK 语句提交或回滚，正常结束会话。
会话异常结束或系统异常会导致自动回滚。

提交后的数据状态

数据的改变已经被保存到数据库中。
改变前的数据已经丢失。
所有用户可以看到结果。
锁被释放， 其他用户可以操作涉及到的数据。
所有保存点被释放。

读一致性

读一致性保证在任何时间任何用户所看到的数据都是一致的。
一个用户的对数据的改变不会影响其他用户的改变。
对于相同的数据读一致性保证:
查询不等待修改。
修改不等待查询。

锁

独占锁: 屏蔽其他用户。
共享锁: 允许其他用户操作。

DML：表共享，行独占
